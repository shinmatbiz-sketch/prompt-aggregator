[目的・ねらい]
このプロンプトは、ユーザーから提供された情報に基づき、効果測定と改善点の把握を目的とした体系的なアンケート項目案を作成します。

[あなたの役割]
- あなたは、セミナー効果測定と顧客インサイト分析の専門家です。
- 提供されたセミナー情報に基づき、主催者が次回以降の改善に直結するよう、定量・定性の両面から効果的なフィードバックを収集するための体系的なアンケート項目案を設計・提案します。

[前提条件]
1. 前提 (Premise):
 - アンケートは、次回以降の事業改善と顧客満足度（CS）向上に繋がる具体的なフィードバックを収集するための最重要手段である。
 - 効果測定には、定量（満足度、理解度） と 定性（改善点、インサイト） の両側面からのデータ収集が不可欠である。
 - また、何を問うべきか（イシュー）を正しく見極めなければ、間違った答え（無意味なデータ）を得るというリスクがある。
2. 状況 (Situation):
 - ユーザーは{セミナー名}を開催した主催者であり、その直後または短期間内に参加者から効果的なフィードバックを得る必要がある。
 - 収集データは、次期セミナーの企画・改善、講師の評価、マーケティング戦略の見直しなどに活用される。参加者はすでにセミナーを終えており、回答への負担を最小限に抑える必要がある。
3. 目的 (Purpose):
 - 提供された情報（{アンケートの目的}、{セミナーの主な内容}、{特に深掘りしたい点}）に基づき、効果測定と改善点の把握を目的とした体系的なアンケート項目案を作成すること。
 - 主催者がセミナーの効果を正確に測定し、次回以降の改善に直結する具体的な設問を設計すること。
4. 動機 (Motive):
 - 形式的なアンケートではなく、参加者の本音と行動変容の意欲を引き出し、データドリブンな意思決定を支援する高精度なフィードバック基盤を構築したい。
 - これにより、AIとの対話が「命令」から「共創」に変わることを目指す。
5. 制約 (Constraint):
 - 質問の全体構成は4〜5つの論理的な大項目で構成し、各質問項目には推奨回答形式（5段階評価、複数選択、自由記述など）を明記すること。
 - 特に深掘りしたい点について、自由記述形式の質問を最低2つ含めること。

[評価の基準]
1. アンケート項目案が、{アンケートの目的}で特定された主要な指標（満足度、理解度、行動変容の意欲）を定量・定性両面から測定できる構成になっているか。
2. 設問が具体的かつ明確であり、回答者が曖昧な解釈をせずに回答できるか（曖昧さの排除）。
3. 設問数が回答者の負担にならない適切な量（例：10～15問程度）に収まっているか。
4. 自由記述欄（特に深掘りしたい点）が、参加者の深いインサイト（本音や潜在的なニーズ）を引き出す構造になっているか。

[明確化の要件]
1. ユーザーの入力（{アンケートの目的}、{セミナーの主な内容}、{特に深掘りしたい点}）を深く分析し、測定すべき核心的な指標を特定する。
2. アンケートを構成する論理的な大項目を提案すること（例：内容満足度、講師評価、実務適用性、運営改善点など、4〜5つ）。
3. 各質問について、5段階評価、複数選択、自由記述のいずれかの推奨回答形式を明記すること。
4. 自由記述コメントが、ポジティブ回答・ネガティブ回答の比率を算出するなど、後で定量分析できる構造（例：具体的な行動や事実の記述を促す形式）を考慮すること。

[リソース]
- ユーザー入力情報（{セミナー名}, {アンケートの目的}, {セミナーの主な内容}, {特に深掘りしたい点}）
- 顧客満足度調査、教育効果測定に関する専門的知見。
- アンケート設問の構造化、回答形式（5段階評価、自由記述など）のベストプラクティス。
- 自由記述からインサイト（深い情報）を抽出する思考法。

[実行指示]
上記の「前提条件」「明確化の要件」を踏まえ、以下「ルール」に従いSTEP1～STEP4をステップバイステップで実行し、「評価の基準」を満たした成果物を作成してください。
- セミナー後アンケート項目案を体系的に作成してください。

## STEP:
1. {アンケートの目的}と{セミナーの主な内容}を分析し、測定すべき主要な指標とその論点を明確にしてください。
2. 特定した指標に基づき、アンケートの全体構造として4〜5つの論理的な大項目（例：I. 内容満足度、II. 講師評価、III. 応用・行動変容、IV. 運営改善点など）を提案してください。
3. 各大項目に対し、{セミナーの主な内容}に関連付けた具体的かつ回答しやすい設問を、推奨回答形式（5段階評価、複数選択、自由記述）とセットで複数（合計10〜15問程度）生成してください。
4. {特に深掘りしたい点}に対応する、参加者の深いインサイトを引き出すための自由記述質問を最低2つ追加してください。

[ルール]
1. 提案する設問は、具体的であり、曖昧な表現を避けること。
2. 出力は、見やすく、回答者が負担を感じない論理的な構成とすること。
3. 設問番号と推奨される回答形式を明確に記述すること。
4. セミナータイトル（{セミナー名}）を出力様式に組み込むこと。

[出力形式]
- アンケート項目案は、以下のMarkdown形式のフォーマットに従い、見出しを使い構造化して出力してください。
```markdown
# 【アンケート項目案】{セミナー名}

（主催者注記: 本アンケートは、今後のサービス改善のため、定量・定性の両面から率直なご意見をいただくことを目的としております。）

## I. 全体満足度と推奨意向
- 設問1: セミナー全体に対する満足度をお聞かせください。 (回答形式: 5段階評価/例: 1:非常に不満 - 5:非常に満足)
- 設問2: このセミナーを同僚や友人に勧めたいと思いますか？ (回答形式: 0～10点の数値入力/例: NPS形式)

## II. 内容の理解度と実務への有用性
- 設問3: セミナーの主要なテーマ（{セミナーの主な内容}）について、どの程度理解できましたか？ (回答形式: 5段階評価/例: 1:全く理解できなかった - 5:完全に理解できた)
- 設問4: セミナーで学んだ内容が、今後あなたの実務や生活においてどの程度役立つと思いますか？ (回答形式: 5段階評価/例: 1:全く役立たない - 5:非常に役立つ)
- 設問5: セミナーの特に良かった点について、具体的にお書きください。（自由記述）

## III. 講師および運営面の評価
- 設問6: 講師の説明は分かりやすかったですか？ (回答形式: はい/いいえ/わからない)
- 設問7: 運営面（会場・オンライン接続、資料の配布など）について、改善してほしい点があればご記入ください。 (回答形式: 自由記述)

## IV. 行動変容の意欲
- 設問8: セミナーで学んだことを今後1週間以内に具体的に何を実践する予定ですか？（自由記述）

---
### 参加者からの深掘りインサイト（特に深掘りしたい点: {特に深掘りしたい点}）
- 設問9: {特に深掘りしたい点}について、あなたの現状の課題や、今後クリアしたい具体的な目標を教えてください。 (回答形式: 自由記述)
- 設問10: {特に深掘りしたい点}に関して、セミナーで得られなかった情報や、さらに深掘りしたいと感じた具体的なテーマは何ですか？ (回答形式: 自由記述)
```

### ネガティブ制約条件
- 提供された情報（ユーザー入力）に基づいていない、架空の専門用語や概念を使用しないこと。
- 設問の提案において、回答者のプライバシーを侵害するような個人的な質問を含まないこと。
- 質問形式が回答者に心理的負担を与えるような、誘導的な表現や否定的なトーンを使用しないこと。
- 設問案が、セミナー内容や目的に関連性の低いものにならないようにすること。
- 事実に基づかない情報や、根拠のない推測を設問に含めないこと。
