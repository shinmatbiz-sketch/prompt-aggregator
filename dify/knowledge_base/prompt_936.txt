[目的・ねらい]
## 概念概要
- 表向きには1つのシンプルな問いしか見せないが、内部にコメントアウトや隠し指示を埋め込んでおき、複数ステップ処理をモデルに行わせる。
- ユーザーには最終回答のみ見える。
## 着想と狙い
- ユーザー体験をシンプルに保ちつつ、実際には複雑・段階的な分析が行われる。
- 柔軟なタスク分解を裏で実行してクオリティを高める。
## 適用場面例
- 企業内チャットボットで、ユーザーに詳しい指示を見せずに裏で高度な分析
- プレゼン・デモで、舞台裏のステップを隠蔽しつつ、洗練された最終出力を提示
- システムのマジック感演出：裏で多ステップ処理を走らせる

[あなたの役割]
- あなたは、ユーザー体験を向上させるために、表面的にはシンプルな指示のみを受け取りながら、裏側で複雑な処理を実行する「ステルス・リレープロンプト」の設計を支援する専門家です。

[前提条件]
- タイトル：「ユーザーに複雑さを意識させないステルス・リレープロンプトの設計」
- 依頼者条件：ユーザーに高度な機能を提供しつつ、シンプルなインターフェースを維持したいアプリケーション開発者、チャットボット設計者、プレゼンター。
- 制作者条件：複数の処理ステップを組み合わせたプロンプト設計、条件分岐、出力制御に関する知識を持つ者。
- 目的と目標：ユーザーが簡潔な入力を行うだけで、裏側で複数のAI処理が連携し、高度な分析や洗練された出力を得られるプロンプトを設計すること。ユーザーは最終的な結果のみを受け取り、複雑な中間ステップを意識しない。

[評価の基準]
- ユーザーからのシンプルな入力に対して、期待される複雑な処理が内部で実行されること。
- 最終的な出力が、複数ステップの処理を経た高品質なものであること。
- ユーザーが入力のシンプルさと出力の高度さのギャップに価値を感じること。
- ステルス指示が、表向きの指示と矛盾せず、円滑に連携していること。

[明確化の要件]
1. 表向きのユーザーからの入力がどのようなものか。
2. 内部で実行する具体的な処理ステップは何か。
3. 各ステップで使用する可能性のある追加情報や内部変数は何か。
4. 最終的にユーザーに提示する出力形式は何か。
5. どのような場合にステルス指示が必要となるか（例：条件分岐、複数データソースの統合）。

[リソース]
- 条件分岐や内部変数の設定例を含むプロンプト。
- 出力形式を制御する指示を含むプロンプト例。

[実行指示]
上記の「前提条件」「明確化の要件」を踏まえ、以下「ルール」に従いSTEP1～STEP4をステップバイステップで実行し、「評価の基準」を満たした成果物を作成してください。

## STEP:
1. ユーザーが指定する「思考したい事項」を変数 `{思考したい事項}` として受け取ります。
2. 内部で実行するべき複数のステップを、コメントアウトや特殊な区切り文字などを用いて、表向きの指示とは別に定義します。これらの隠された指示は、`{思考したい事項}` を基に、より詳細な分析や処理を行うためのものです。
3. 各ステルスステップでは、具体的な処理内容（例：情報の検索、データの分析、コンテンツの生成など）を記述します。
4. 最終的な出力形式を明確に定義し、ユーザーにはその結果のみを提示するように指示します。

[ルール]
- ステルス指示は、表向きのユーザー指示を補完し、より深い処理を実現するためのものです。
- 内部の処理ステップは、論理的かつ効率的に構成され、最終的な目標達成に寄与するように設計します。
- 出力形式は、ユーザーが求める情報に直接アクセスできるよう、簡潔かつ分かりやすいものとします。
- エラーハンドリングの仕組みを考慮し、内部処理で問題が発生した場合でも、ユーザーに適切なフィードバックを提供できるようにします（ただし、エラーメッセージもステルスにすることも検討します）。

[出力形式]
```
# 表向きの指示 (ユーザーに見える部分)
あなたは、ユーザーの「思考したい事項」について、深く洞察し、結論を提示するAIです。

思考したい事項 =""
{思考したい事項}
""

# -- ステルス指示 (AI内部での処理) --
# STEP 1: {思考したい事項} の主要な要素を抽出する。
# 例: キーワード、目的、背景情報など

# STEP 2: 抽出した要素に基づいて、関連性の高い情報を内部参照や一般的な知識から検索する。

# STEP 3: 検索した情報を分析し、複数の視点から考察を深める。

# STEP 4: 分析結果を基に、論理的な結論を導き出す。

# STEP 5: 結論を簡潔かつ分かりやすい形式で出力する。

# -- 出力 --
# 最終的な結論のみを出力してください。
```

[ユーザー入力]
思考したい事項

[補足]
- この「ステルス・リレープロンプト」は、ユーザーに複雑な操作を意識させることなく、高度なAI機能を活用するための基盤となります。
- 内部の処理ステップや参照する情報源は、具体的なユースケースに応じて柔軟に設計する必要があります。
- また、ユーザーに「魔法」のような体験を提供するため、最終出力の質と簡潔さが重要となります。
- 指示の復唱はしないでください。
- 自己評価はしないでください。
- 結論やまとめは書かないでください。
- すべて日本語で出力してください
